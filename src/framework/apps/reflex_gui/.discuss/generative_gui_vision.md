### 动态渲染引擎
基于`stream_gui_test.py`的验证，使用`rx.match` + `rx.foreach`实现动态组件渲染：

```python
def render_component(comp: dict):
    return rx.match(
        comp["type"],
        ("markdown", rx.markdown(comp["content"])),
        ("card", rx.card(...)),
        ("divider", rx.divider()),
        ("image", rx.image(src=comp["src"])),
        ("chart", render_plotly_chart(comp)),
        rx.text("未知组件")
    )
```

## 关键决策点

### 1. 组件复杂度边界
- **简单组件**：直接使用Reflex原生组件
- **中等组件**：组合多个基础组件
- **复杂组件**：需要自定义实现或第三方集成

### 2. 数据绑定策略
- **静态数据**：直接嵌入在组件定义中
- **动态数据**：通过API接口实时获取
- **实时流**：WebSocket或Server-Sent Events

### 3. 交互逻辑处理
- **客户端交互**：通过Reflex的`on_click`等事件
- **服务器端交互**：通过API调用后端逻辑
- **混合模式**：客户端触发 + 服务器端处理
